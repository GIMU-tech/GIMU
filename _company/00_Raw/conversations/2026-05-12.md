### 🎨 디자인 브리프: AI Response Feedback System

#### 1. 디자인 전략 (Design Strategy)
- **목표:** 사용자가 AI의 작업 상태(생각 중, 작성 중, 완료, 오류)를 직관적으로 인지하여 '기다림의 불확실성'을 해소함.
- **핵심 가치:** 투명성(Transparency), 즉각성(Immediacy), 신뢰도(Reliability).

#### 2. 시각적 컨셉 (Visual Concept)
| 요소 | 컨셉 A: Minimal Pulse (미니멀 펄스) | 컨셉 B: Dynamic Flow (다이내믹 플로우) |
| :--- | :--- | :--- |
| **핵락 키워드** | 정적, 정갈함, 텍스트 중심 | 역동적, 흐름, 그래디언트 |
| **상태 표현** | 텍스트 커서의 미세한 깜빡임 + 은은한 Glow | 선(Line)의 흐름과 입자(Particle)의 움직임 |
| **타깃 UI** | 깔끔한 코딩 환경(VS Code 스타일) | 미래지향적 인터페이스(Modern Web 스타일) |

#### 스텝 3. 상태별 디자인 가이드 (Draft)

**A. [Thinking] AI가 질문을 분석 중일 때**
- **Visual:** 텍스트 입력창 하단에 아주 가느다란 `Progress Line`이 왼쪽에서 오른쪽으로 부드럽게 움직임 (Linear Motion).
- **Interaction:** 커서가 `_`에서 `|`로 깜빡이며 '대기 중'임을 암시.

**B. [Generating] AI가 답변을 작성 중일 때**
- **Visual:** 글자가 나타나는 속도에 맞춰 `Pulse` 효과가 발생. (글자가 써지는 위치에 빛이 모이는 느낌)
- **Interaction:** 텍스트가 생성되는 동안 스켈레톤(Skeleton) UI가 아닌, 실제 텍로딩(Typing) 효과와 함께 미세한 그라데이션이 흐름.

**C. [Error/Idle] 응답이 멈췄거나 오류가 났을 때**
- **Visual:** `Error` 발생 시 붉은색 `Glow`가 경계선에 나타남.
- **Interaction:** 응답이 일정 시간(예: 10초) 동안 없으면, '연결 확인' 버튼이 나타나거나 경고 아이콘이 부드럽게 나타남.

#### 3. 컬러 & 타이포 가이드 (Proposed)
- **Primary Color:** `#6366F1` (Indigo Blue - 지적이고 신뢰감 있는 색상)
- **Secondary Color:** `#10B981` (Emerald - 성공/완료를 나타내는 색상)
- **Error Color:** `#EF4444` (Rose - 즉각적인 주의를 끄는 색상)
- **Typography:** `Inter` 또는 `JetBrains Mono` (가독성이 높은 코딩용 폰트 권장)
